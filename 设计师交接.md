## 3. 如何打开与演示

### 本地预览

- 直接双击 `index.html`，或
- 本地静态服务器打开（避免部分浏览器对 `file://` 的限制）

### 演示路径建议

1. **New Design** → 展示完整 Dashboard（默认）
2. 切换 **中文** → 展示双语能力
3. 浏览器缩窄至手机宽度 → 展示抽屉侧栏 + 表格横滚
4. **Original** → 与旧版对比
5. **Specs** → 设计规范速览
6. **Themes** → 说明 Vivid Sky 为生产色，其余为备选

### 四个 Tab 命名（已优化）

| 内部 ID | 显示名 EN | 显示名 ZH |
|---------|-----------|-----------|
| `before` | Original | 原版 |
| `updated` | New Design | 新设计 |
| `design-system` | Specs | 设计规范 |
| `tweak` | Themes | 配色 |

## 6. 已知限制（原型性质）

| 限制 | 说明 |
|------|------|
| 展示型交互 | 除侧栏导航、语言切换、图表 hover 外，多数按钮不可点击（`tabindex="-1"`） |

| 无 Figma | 视觉稿即 HTML；后续若建 Figma 需以此为准对齐 |

## 8. 关键设计决策（勿随意改动）

| 决策 | 原因 |
|------|------|
| Vivid Sky 为生产色 | 用户明确确认；Specs 与 New Design 默认均为此色 |
| 图表单色系 + 一处 coral accent | 保持专业感，强调「关键数据点」 |
| Teal / Amber / Coral 仅用于 Status Tag | 不与图表色混淆 |
| 无默认卡片投影 | 近白底上靠 1px 边框分层；hover 仅 border tint |
| Welcome 独立渐变卡片 | 与 KPI 白卡区分层级，非又一扁平白卡 |
| 费用卡锁图标仅顶部 chip | 避免行内重复锁造成噪音 |
| 进度条 = paid / pending | 非金额进度，实色 + 斜纹区分 |
